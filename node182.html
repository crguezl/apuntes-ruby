Subclases y Herencia

  • Si, cuando se define una clase no se especifica una superclase entonces nuestra clase hereda implíctamente de Object<#6953#><#6953#>.

  • The fact that <#6954#>classes may have multiple subclasses but only a single superclass (single inheritance) means that they can be arranged in a tree structure<#6954#>, which we call the Ruby class hierarchy.
  • The BasicObject<#6955#><#6955#> class is the root of this hierarchy, and every class inherits directly or indirectly from Object<#6956#><#6956#>.
  • The descendants of a class are the subclasses of the class plus the subclasses of the subclasses, and so on recursively.
  • The ancestors of a class are the superclass, plus the superclass of the superclass, and so on up to Object.
  • The syntax for extending a class is simple. Just add a 693 character and the name of the superclass to your class statement.

    We then create a subclass using class 694. The 695 notation means we’re creating a subclass of the thing on the right; the fact that we use less-than presumably signals that the child class is supposed to be a specialization of the parent.
